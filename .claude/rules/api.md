  We use the following tools to interact with the project's API:

- Orval (https://orval.dev/) to generate functions and types
- TanStack Query (integrated with Orval)

## Rules

- **AVOID** transforming entire pages into Client Components.
- Prefer to fetch data from Server Components whenever possible.
- **ALWAYS** use the functions present in @lib/api/generated to fetch data from Client Components.
- **ALWAYS** use TanStack Query to perform all data fetching on the client-side (Client Components) and, for this, **ALWAYS** use the hooks present in @lib/api/generated.
- If you need a function that is not present in @lib/api/generated, run the command `npx orval` to generate the files again. If the function is not present after executing the command, INTERRUPT your response and notify the user.
- When calling `authClient`, **NEVER** place it inside a `try, catch` block. **ALWAYS** destructuring the `error` that comes from your result and handle it correctly. Example: `const { error } = await authClient.changePassword({})`
- **ALWAYS** call the synchronous variation of the mutation when using hooks from @lib/api/generated. In these cases, **ALWAYS** handle the success and error cases using the `onError` and `onSuccess` parameters. Example:

  ```tsx
  const { mutateAsync: createGateway, isPending: isCreating } =
    useCreateStorePaymentGatewayIntegration();

  const onSubmit = (payload) => {
    createCondition(
      {
        storeId,
        data: payload,
      },
      {
        onSuccess: () => {
          toast.success("Shipping configuration created successfully!");
          queryClient.invalidateQueries({
            queryKey: getGetStoreShippingCostConditionsQueryKey(storeId),
          });
          onClose();
        },
        onError: (error) => {
          if (
            error.response?.data.code === "ShippingCostConditionConflictError"
          ) {
            return toast.error(
              "There is already an active shipping configuration for this period."
            );
          }
          const errorMessage =
            error?.response?.data?.message ||
            "Error creating shipping configuration.";
          toast.error(errorMessage);
        },
      }
    );
  };
  ```

## Data Fetching: Server-side e Client-side

**PRIORITIZE** fetching data on the server-side using `fetch` and using the response result as `initialData` in the hooks generated by TanStack Query.

- When fetching data on the server-side, **ALWAYS** use the functions exported from @app/\_lib/api/fetch-generated/index.ts.

- When fetching data on the client-side, **ALWAYS** use the hooks exported from @lib/api/rc-generated/index.ts.

Example:

```tsx
// page.tsx (Server Component)
import { getHomeDate } from "@/lib/api/fetch-generated";

const Home = async () => {
  const data = await getHomeDate(new Date());

  return <ClientComponent data={data} />;
};

export default Home;

// client-component.tsx (Client Component)
import { useGetHomeDate } from "@/lib/api/rc-generated";

export const ClientComponent = (props) => {
  const result = useGetHomeDate(props.today, {
    query: { initialData: props.data },
  });

  return <></>;
};
```
